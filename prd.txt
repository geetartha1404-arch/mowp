1. Product Overview
MOWP is an AI-assisted personal work management system built as a multi-tenant SaaS. Each user gets their own isolated workspace where they can capture tasks, notes, and ideas; receive AI-driven prioritization; plan their week; and get automated daily and weekly review summaries.

The product is not a universal life OS. Its job-to-be-done is narrow and clear: reduce task fragmentation and planning overhead for solo knowledge workers by automating the capture → classify → plan → review loop.

Positioning: "An AI-assisted work OS that turns scattered tasks into a clear daily plan — automatically."

1.1 Target User
Solo knowledge workers: developers, designers, PMs, consultants, writers
People managing multiple concurrent responsibilities without a team system
Individuals who want AI to do the prioritization thinking, not just suggest completions

3.2 Multi-Tenant Design
Multi-tenancy is implemented via Supabase Row Level Security (RLS). Every table has a user_id column. RLS policies enforce that users can only read and write their own rows — at the database level, not just the application level. No custom tenant logic is needed in Next.js.

Every Supabase table has: user_id UUID NOT NULL REFERENCES auth.users(id). Every RLS policy uses: USING (user_id = auth.uid()).

4. Feature Specifications
4.1 Authentication
Supabase Auth handles all authentication. Next.js uses the @supabase/ssr package for server-side session management.

Sign up with email + password
Email verification on sign-up
Login / logout
Password reset via email
Session persistence via HTTP-only cookies (Supabase SSR)
Protected routes — all app pages redirect to /login if unauthenticated

4.2 Inbox Capture
The primary entry point. Users add any work item — task, idea, note, request — in one action. After save, an AI classification job runs asynchronously and updates the item within seconds.

Required field: title
Optional: description, type, due date, tags
After save: item appears in inbox with status "processing"
AI classification completes: category, priority, summary, next action populated
Item moves to "inbox" status, ready for planning

4.3 AI Classification Flow
Model: google/gemini-2.0-flash-exp via OpenRouter. Prompt version logged on every call.

On item creation, an Edge Function is triggered with the item title, description, type, and due date. The AI returns a structured JSON response:

AI Output Field
Description
category
Short label: "Engineering", "Strategy", "Admin", "Learning", etc.
priority
One of: critical / high / medium / low / someday
effort_estimate
low / medium / high
summary
One-sentence plain-English description of the task
next_action
The single most concrete next physical action
tags
Up to 3 suggested tags

4.4 Weekly Planning Board
View: Kanban-style or list view, grouped by status
Columns: Inbox → Planned → In Progress → Done
Filter by: week, priority, category, tag
Drag to reorder within columns
Inline edit: due date, effort, priority, status
AI next action visible on each card

4.5 Daily Digest
Auto-generated each morning at a user-configured time (default: 07:00 local). Triggered by a Supabase Edge Function cron job.

Top 3–5 priority tasks for today (AI-selected)
Overdue items list
Blockers identified by AI from item descriptions
AI-written focus summary: 2–3 sentences
Visible on dashboard homepage and notification channel (email optional)

4.6 Weekly Review
Auto-generated every Sunday at 18:00. Triggered by cron Edge Function.

Completed count vs planned count
Overdue and carry-forward items
AI-written review: what got done, what slipped, why
2–3 AI improvement suggestions for next week
Historical reviews accessible via review archive page

6. AI Behavior Specification
The AI should make structured work decisions, not act as a general-purpose chatbot.

6.1 What the AI Does
Use Case
Trigger
Output
Classification
Item created
Category, priority, effort, summary, next action

Cron 07:00
Top tasks, blockers, focus summary
Weekly Review
Cron Sunday 18:00
Stats summary, improvement suggestions

6.2 Prompt Design Rules
All prompts request JSON output — parse, never eval
Each prompt has a version string stored in prompt_logs
Prompts constrained: AI returns only structured fields, no prose unless asked
Failure handling: if AI response fails to parse, item is flagged for manual review
No user-facing "chat" interface — AI is a background service
